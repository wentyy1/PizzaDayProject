# ADR 0001: Вибір архітектурного стилю - Модульний моноліт

## Контекст
Проєкт "Pizza Day" знаходиться на початковій стадії розробки. Команда складається з 1-2 розробників. 
Потрібно швидко запустити MVP з базовим функціоналом замовлення піци.

## Рішення
Обрано **модульний моноліт** з чіткими межами між доменними контекстами.

## Альтернативи
**Мікросервіси** розглядалися, але відхилені через:
- Велика операційна складність для маленької команди
- Необхідність інфраструктури для комунікації між сервісами
- Повільніша розробка на початковому етапі

## Наслідки
### Позитивні
- Швидший старт і розробка
- Простіше тестування та дебаггінг
- Менше операційних витрат

### Негативні
- Потенційні проблеми з масштабуванням
- Ризик "big ball of mud" без дотримання архітектури

## Статус
Прийнято - 2024

## Умови перегляду
Розглянемо перехід на мікросервіси при:
- Команда збільшиться до 3+ розробників
- Потреба незалежного масштабування окремих компонентів
- Необхідність різних технологій для різних доменів
